# 贝贝外刊每日精读网站需求

本文档整理自 Codex 对话“汇总贝贝外刊需求”及其后续修订。

## 1. 核心目标

将 `/Users/apple/Downloads/贝贝外刊` 中持续更新的 PDF 讲义：

- 自动监控并增量处理，避免重复收录；
- 汇总到统一的 `$beibei` skill；
- 生成按日期组织的外刊学习网站；
- 推送 GitHub，并通过 Render 自动部署。

处理链路：

```text
贝贝外刊 PDF
→ 增量更新 $beibei skill
→ 重新生成时间轴首页和每日页面
→ 推送 GitHub main
→ Render 自动部署
```

## 2. 网站结构

### 时间轴首页

- 按日期倒序展示每天的外刊讲义；
- 展示期数、文章标题、摘要、页数、段落数、词条数和长难句数量；
- 可以进入对应日期的详情页；
- 当前收录 2026-06-11、2026-06-12、2026-06-18 三期。

### 每日详情页

文章原文开始前，依次展示“阅读导入”：

- 一段中文背景介绍，说明人物、事件与报道语境；
- 一段中文内容简介，梳理文章主要内容和论证路线；
- 一段基于原文核心信息、参考 PET / CEFR B1 水平改写的英文版本，使用常用词和较短句式。

随后展示：

- 英文原文；
- 对应中文译文；
- 重点单词、音标、词性、英文解释、中文释义和例句；
- 长难句分析。

不直接照搬 PDF 中的以下教学附录：

- 文章结构；
- 课后作业。

## 3. 原文与译文

- 原文与译文必须按段正确对应；
- 不能用背景材料或推测内容冒充译文；
- 需要兼容 PDF 双栏和文本抽取顺序异常；
- 对抽取结果进行校验，尤其关注尾段译文；
- 每个原文和译文段落均可点击放大；
- 放大浮窗以大字号显示纯正文，不带词义悬浮内容；
- 支持关闭按钮、点击遮罩和 `Esc` 关闭；
- 支持键盘聚焦与回车打开，关闭后焦点返回原段落。

## 4. 词汇准确性

- 单词释义必须结合文章语境，不能只套用常见词义；
- OCR 拼写错误需要识别和纠正；
- 防止 PDF 双栏或词条边界异常把整段译文串入词义或例句；
- 生成器应拦截超长异常释义，提示人工校对；
- 已核验修复 `ayatollah`、`emblem`、`obscene`、`collateral`、`congresswoman`、`stretch` 和 `contentment`；
- 当前三期共 183 个词条。

## 5. 词汇交互

- 原文中的重点词添加细红色下划线；
- 鼠标悬停时显示音标和中文释义；
- 同一个词在一篇文章中只在第一次出现时标注，后续出现保持普通正文；
- 保留独立词汇列表和搜索；
- 英文解释需要足够清晰，使用较大字号、舒适行高和高对比度；
- 点击词卡或放大按钮可查看完整词条；
- 每个词条可以收藏和取消收藏；
- 顶部提供“我的收藏”入口和数量；
- 收藏使用浏览器本地存储，跨日期、刷新后仍保留；
- 收藏夹中可以搜索、再次放大或移除词条；
- 不保留用途不清晰的右上角“已掌握”方块。

## 6. 长难句分析

- 忠实保留 PDF 原版格式与顺序；
- 使用“粗体完整长句 → 编号分析小节 → 逐行语法成分说明”的层级；
- 两条长难句分别展示为独立分析卡；
- 空行、换行和跨页续接以讲义原版版面为准；
- 不按“主句、从句、语法点”等关键词自行拆段；
- 不把分析压缩为一整段；
- 不得混入文章译文。

## 7. 视觉与响应式

- 整体采用适合长时间阅读的浅色纸张风格；
- Header 不使用黑色或突兀的高饱和颜色，配色与正文协调；
- 长难句使用白底、深色文字；
- 避免绿色浅字和内容拥挤；
- 同时适配桌面与手机；
- 页面及词义悬浮层不能产生横向溢出。

## 8. 更新与部署

- 生成器：`scripts/build_beibei_daily_site.py`；
- GitHub：`twfour/beibei-word-site`；
- Render：`https://beibei-word-site.onrender.com/`；
- 部署形式：Python Web Service；
- 健康检查：`/api/health`；
- OSS/CDN 不再是当前主线。
