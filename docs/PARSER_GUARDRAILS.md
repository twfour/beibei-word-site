# 贝贝外刊解析错误记录与防回归规则

本文记录已经发现的 PDF 解析失败模式。修改生成器时必须保留对应防护，新增 PDF 后必须执行文末检查。

## 1. 长难句正文被吞进原句

### 现象

2026-06-12 第一条长难句在网页中全部进入粗体原句框，分析区为空。

### 原因

讲义中的分析标题不统一：

- 第一题使用“第一部分：/第二部分：/第三部分：”；
- 第二题使用“1. 主语部分/2. 插入语”。

旧解析器只把以 `1.` 或 `1、` 开头的行视为分析起点，因此无法识别“第一部分：”，将整段分析误当成原句。

### 防回归规则

- 同时识别数字标题、中文“第…部分”、主句部分和整体分析标题；
- 每条长难句的分析正文不得为空；
- 原句字段超过 1200 字符视为结构异常；
- 任一检查失败时立即停止生成，不允许输出错误 HTML。

## 2. 词义与文章译文粘连

### 现象

`ayatollah`、`emblem`、`obscene`、`collateral`、`congresswoman`、`stretch` 的释义曾混入整段文章译文。

### 原因

PDF 双栏阅读顺序、跨栏对象顺序或缺少词条例句分隔符 `•`，使文本抽取器无法判断词条结束位置。此问题不是普通 OCR 拼写错误。

### 防回归规则

- PDF 原页是唯一事实依据；
- 已核验异常保留在 `VOCAB_CORRECTIONS`；
- 中文释义包含超过 80 个汉字，或超长且带多个句末标点时，`suspicious_definition()` 必须阻止生成；
- 不得仅凭网页结果猜测词义或自动截断。

## 3. `contentment` 例句边界错位

### 现象

词义本身正确，但第一条例句后粘入文章译文，第二条例句丢失。

### 原因

PDF 把段落译文排在两条例句之间，线性文本抽取破坏了视觉上的词条边界。

### 防回归规则

- `contentment` 的释义和两条例句使用原页核验后的修正值；
- 检查例句字段是否出现与词条无关的长段中文；
- 词义正确不代表整个词卡正确，释义、英文解释和例句必须分别检查。

## 4. 长难句版式被自行拆分或压平

### 现象

长难句分析曾被网页按“主句/从句/语法点”等关键词自行切段，也曾被压成单一段落。

### 原因

使用普通文本抽取结果推测版式，没有读取 PDF 的实际行结构和跨页续接。

### 防回归规则

- 长难句必须使用 `pdfplumber.extract_text(layout=True)`；
- 空行、编号、逐行说明和跨页顺序以 PDF 版面为准；
- 每条完整原句与其分析分别放入独立卡片；
- 不用关键词重新发明段落层级。

## 5. 原文与译文错位

### 风险

双栏 PDF、尾段和跨页内容可能改变文本抽取顺序，导致原文与译文数量或顺序不一致。

### 防回归规则

- 原文和译文必须逐段对应；
- 背景资料、文章结构和课后作业不得充当译文；
- 无法可靠识别时显示缺失提示，绝不生成推测译文；
- 特别检查最后一段和跨页段落。

## 6. OCR 拼写判断

### 防回归规则

- 遇到疑似拼写错误，先查看 PDF 原页；
- 区分真正的 OCR 拼写错误与“正确单词 + 错误字段边界”；
- 修复应优先落在生成器或源数据修正规则中，不直接手改生成 HTML。

## 每次生成后的必检项

1. 三期文章数量、段落数、词条数和长难句数保持预期；
2. 每条长难句同时具有非空原句与非空分析；
3. 原句字段没有混入“第一部分、主句、语法点”等分析词；
4. 释义和例句没有混入整段文章译文；
5. 原文与译文段落数量一致；
6. 桌面与 390px 手机宽度均无横向溢出；
7. 修改生成器后重新生成所有日期页，不直接修生成文件。
